## 2. 达标 idea

**Robot Data Unit Tests: 面向 VLA 训练集的 action/metadata/视觉-运动一致性 linter**

一句话故事: 把 OXE/LeRobot/DROID 这类机器人示教集当作可测试的软件构件,在 VLA 训练前自动运行 action 单位、坐标系、gripper polarity、timestamp、视觉-运动闭环和任务文本一致性测试,输出可复现的 lint failure、最小修复和 cleaned-vs-raw 训练收益。

核心机制:
- 数据不变量: action 积分后的末端轨迹应与 recorded state / image-space end-effector motion 在时序上闭合; gripper action 的符号和幅值应与可见夹爪开合、接触事件和 state channel 同步; timestamp 与速度/加速度分布不得出现跨 episode 的不可解释尖峰; instruction 中的主 object 应与实际移动/接触 object 对齐。
- 差分测试: 对同一数据源运行 raw metadata、候选 unit/frame/polarity/timing transform、视觉反演轨迹三套解释,选择能同时最小化 kinematic residual、image-motion residual 和 gripper residual 的解释; 多解释都能成立时标为 ambiguous,不自动修。
- 基准构造: 在 OXE/LeRobot 子集注入可控 corruptions,评估检测率、误报率和最小修复正确率; 再对真实公开数据集做 blind audit,训练小型 Diffusion Policy / OpenVLA-style adapter 比较 raw、drop-bad、auto-fix 三种数据路径。

### 定向查重记录

检索组:
- 问题表述: `robot dataset validation`, `robot demonstration consistency`, `VLA data linter`, `action units coordinate frames robot demonstrations`, `Open X-Embodiment metadata action gripper`。
- 方法机制: `influence functions robot data curation`, `suboptimal demonstration learning`, `action metadata certificate VLA`, `data management large robot datasets`。
- 相邻工具/非论文占位: LeRobot 数据栈、Robo-DM、OXE/OXE-AugE、Isaac/合成数据管线。

最相近工作:

| 工作 | 相似点 | 差异 |
|---|---|---|
| QoQ / ATHENA | 都做 robot data curation,目标是更少数据训练出更好 policy。 | 它们从训练 loss/influence 反推样本效用;本 idea 在训练前运行语义单元测试,定位 unit/frame/polarity/timing 等可解释错误,可与 influence curation 串联。 |
| Ambient Diffusion Policy | 都面对低质量混合集数据。 | ADP 学会容忍 suboptimal data;本 idea 直接检测和修复数据语义错误,目标是让数据进入任何 policy 前先过一致性测试。 |
| Same Weights, Different Robot | 都把 action metadata 视为 executable policy 的一部分。 | 该工作聚焦部署端 metadata mismatch certificate;本 idea 聚焦训练集内部跨来源示教的 lint benchmark、自动修复和 cleaned-vs-raw 训练收益。 |
| OXE-AugE / Open X-Embodiment | 都服务跨 embodiment 泛化。 | 它们扩大数据覆盖和格式聚合;本 idea 检查聚合后轨迹语义是否真的可训练。 |
| Robo-DM / LeRobot | 都属于 robot data infrastructure。 | 它们优化采集、存储、加载和复现实验;本 idea 增加类似软件测试的 semantic correctness layer。 |

结论: 本轮检索没有发现直接以 "robot demonstration semantic lint/unit tests" 为贡献主体的工作。最接近的是部署端 ExecSpec 和 influence-based curation,但二者都没有覆盖训练集级别的 action/metadata/视觉-运动一致性测试。

#### 8. Verdict

**Strong Accept**

优先执行点:
1. 定义 8-12 个 semantic unit tests: unit/frame、delta-vs-absolute、gripper polarity、timestamp jitter、state-action closure、image-motion closure、instruction-object contact、episode reset boundary。
2. 构造 corruption benchmark: 在干净子集注入单一错误与组合错误,报告 detection、false positive、auto-fix accuracy。
3. 做 raw/drop/fix 三路训练验证: 小模型先证明 lint failure 与 policy failure 有因果关系,再扩到 OpenVLA-style adapter。
