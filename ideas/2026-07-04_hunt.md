# 2026-07-04 Hunt

## 1. 关键文献

- [Open X-Embodiment: Robotic Learning Datasets and RT-X Models](https://arxiv.org/abs/2310.08864): 将多机器人数据转换到统一格式,证明跨 embodiment 训练有效,但统一格式不等于 action 语义完全兼容。
- [Vision-Language-Action in Robotics: A Survey of Datasets, Benchmarks, and Data Engines](https://arxiv.org/abs/2604.23001): 把 VLA 进展的核心瓶颈定位到 datasets / benchmarks / data engines,明确 representation alignment 与 scalable data generation 仍未解决。
- [A Survey on Vision-Language-Action Models: An Action Tokenization Perspective](https://arxiv.org/abs/2507.01925): 系统梳理 action token 类型,说明 VLA 的关键设计差异集中在 action 如何表示。
- [FAST: Efficient Action Tokenization for Vision-Language-Action Models](https://arxiv.org/abs/2501.09747): 用 DCT 压缩高频动作序列,并发布 FAST+ universal tokenizer；解决 tokenization 效率,但不检查 token 背后的控制语义契约。
- [SpatialVLA: Exploring Spatial Representations for Visual-Language-Action Models](https://arxiv.org/html/2501.15830v1): 用 Ego3D 编码和 adaptive action grids 处理跨机器人空间对齐,代表“学一个统一 action 表示”的路线。
- [Latent Action Diffusion for Cross-Embodiment Manipulation](https://arxiv.org/html/2506.14608v3): 通过 latent action space 做跨 embodiment manipulation,说明 action-space mapping 已是热点,但仍偏表示学习而非兼容性审计。
- [Metamorphic Testing of Vision-Language Action-Enabled Robots](https://arxiv.org/abs/2602.22579): 用 metamorphic relations 缓解 VLA 测试 oracle 问题,与本 idea 的动态测试相邻；差异在于本 idea 的对象是 action ABI 契约和数据/模型接口语义。

## 2. 达标 idea

**Action ABI: 给 VLA 动作空间建立可检查的语义契约。**

一句话故事：跨 embodiment VLA 现在把不同机器人数据“转成统一格式”后直接训练,但 frame、控制频率、delta/absolute pose、EEF/joint/control mode、gripper polarity、delay compensation 等底层语义常被隐式假设；Action ABI 将这些语义显式化为机器可检查契约,并用静态 lint + 小规模动态 compatibility tests 在训练前发现不可混合、需 adapter、可安全复用的数据边界。

核心贡献形态：

- 新问题定义：VLA 数据兼容性不是 file schema 问题,而是 action ABI 语义契约问题。
- 工具/benchmark：对 Open X-Embodiment / LeRobot / DROID / LIBERO 风格数据建立 action ABI schema、lint rules、compatibility score。
- 方法：自动推断缺失 ABI 字段,检测单位/坐标系/控制模式/频率/夹爪语义错配；给出最小 adapter 或拒绝混合。
- 初步实验：同一 VLA/tokenizer 在“ABI-clean mixture”和“ABI-contaminated mixture”上训练或微调,观察成功率、action reconstruction、cross-robot transfer 的差距。

### 定向查重记录

| 最相近工作 | 相近点 | 差异 |
|---|---|---|
| [Open X-Embodiment](https://arxiv.org/abs/2310.08864) | 统一多机器人数据格式,训练 RT-X | 主要是数据汇聚与格式统一,没有提出 action ABI 契约、静态 lint 或兼容性边界。 |
| [FAST](https://arxiv.org/abs/2501.09747) | universal action tokenizer,覆盖多种 action sequences | 优化 tokenization 和训练效率,默认输入动作语义已可混合；Action ABI 检查 tokenization 之前的语义可交换性。 |
| [SpatialVLA](https://arxiv.org/html/2501.15830v1) | 处理跨机器人空间/action 对齐 | 通过 representation 学习消化差异；Action ABI 先暴露哪些差异不应被盲目消化。 |
| [Latent Action Diffusion](https://arxiv.org/html/2506.14608v3) | 学 latent action space 做跨 embodiment transfer | 需要 action-space correspondences；Action ABI 可作为 correspondence 质量和适用边界的前置审计。 |
| [Metamorphic Testing of VLA-enabled Robots](https://arxiv.org/abs/2602.22579) | 用 metamorphic testing 找 VLA 执行失败 | 面向输入扰动和执行 oracle；Action ABI 面向数据/模型动作接口,覆盖训练前静态检查与最小动态 compatibility probe。 |

查重结论：未发现直接等价工作。相邻工作集中在统一格式、action tokenization、latent action 表示、VLA 行为测试；缺口是“统一格式之下的 action 语义契约是否兼容”。

### 完整评审表

#### 1. First impression

- Paper type: Novel Problem + Benchmark/Tooling Method.
- One-sentence story: VLA 领域正在扩大跨机器人数据混合,但缺少类似 ABI 的动作语义契约层,导致不可见的数据污染和错误迁移。

#### 2. Fatal-flaws audit

| # | Flaw | Severity | Defense |
|---|---|---|---|
| 1 | 可能被认为只是数据清洗工程 | MAJOR | 把贡献限定为可复现的 ABI taxonomy、自动检测规则、compatibility score,并证明 ABI 污染会系统性伤害 VLA/tokenizer。 |
| 2 | 可能被 action tokenization / latent action work 视为已解决 | MAJOR | 明确层次差异：tokenizer 学表示,ABI 检查物理控制语义；实验展示同一 tokenizer 在 ABI-clean 与 contaminated mixture 下的差距。 |

#### 7. Integrity gate result

- Gate 1-7: pass. Novelty claim 已用 5 个相近方向定向查重；Strong Accept 依赖问题定义与实验切片收窄,不是泛泛的数据清洗。

#### 8. Verdict

**Strong Accept**

理由：该 idea 把 VLA 数据基础设施中一个被格式统一掩盖的底层问题独立成研究对象,相邻工作没有直接覆盖 action ABI 契约。它的投稿形态可以很克制：问题定义 + taxonomy + lint/probe 工具 + controlled contamination 实验。若实验证明 ABI violation 会稳定损害 tokenizer/VLA transfer,clear accept 概率高。
