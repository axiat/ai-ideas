Nearest Work:
- MIKASA-Robo: Memory, Benchmark & Robots | https://arxiv.org/abs/2502.10550 | 2025-02, v2 in 2025-06 | Provides 32 tabletop-manipulation tasks in ManiSkill3 that isolate four memory categories: object, spatial, sequential, and capacity. It mainly compares RL baselines such as PPO-MLP, LSTM, SAC, TD-MPC2, and DT; v2 section 6.4 adds 2 off-the-shelf baselines, Octo and OpenVLA, on selected tasks | Substantially occupies isolated task families and a four-part memory taxonomy. The categories overlap semantically on 3/4 axes; procedural memory is unique to this idea. It does not build a matrix of memory representations on one VLA backbone, and its short tasks lack high-quality demonstrations for imitation.

Strongest Counterexample: MIKASA-Robo (2502.10550) — Published 11 months earlier, it already provides isolated manipulation task families and a four-part memory taxonomy, structurally matching the first half of the headline. Its v2 does not cover long-horizon tasks with imitation-quality demonstrations or a 10+ memory-variant matrix on one VLA backbone. The remaining distinction lies in evaluation target, VLA and imitation rather than RL, and the variant × memory-type matrix; whether that reaches clear accept depends on the value of those unoccupied dimensions.
